# Политика конфиденциальности

*Обновлено: 2026-05-15*

Noctis делает одну вещь — маршрутизирует трафик браузера через прокси-серверы, которые вы настроили. Эта страница объясняет, какие данные Noctis трогает, что с ними делает и чего явно не делает.

Актуальная версия документа: <https://noctis.c0nn3ct.xyz/ru/privacy/>.

## Что Noctis хранит

Расширение хранит следующее в локальном storage браузера (`chrome.storage.local`):

- Список добавленных прокси-серверов.
- Профили маршрутизации и правила.
- Subscription URL и расписание обновлений.
- Настройки UI (тема, закрепление side panel, уровень логов и т.д.).

Нативный хелпер (`noctis-host`) пишет конфиг sing-box в каталог пользовательских данных ОС. Файл зеркалит те же серверы и правила.

## Что Noctis отправляет в сеть

Единственный сетевой трафик, который Noctis генерирует сам:

1. **Проксированный трафик** — запросы вашего браузера, направленные через прокси-серверы, которые вы настроили. Noctis не выбирает назначения; их выбираете вы.
2. **Обновления подписок** — если вы добавили subscription URL, Noctis периодически забирает его, чтобы обновить список серверов. Никакой subscription URL не добавляется без вашего ведома.
3. **Проверка IP** — только при активном подключении, по запросу, запрос к api.ipify.org через активный прокси, чтобы popup показал ваш видимый exit IP. Это подтверждение работы прокси; не запускается при отключении, никогда не раскрывает реальный IP этому сервису.

Всё. Никакого SDK аналитики, отчётов о крашах, удалённой конфигурации, opt-in/opt-out телеметрии, аккаунтов.

## Что Noctis не делает

- Не отправляет историю браузинга, список серверов, настройки или какой-либо идентификатор разработчику. Единственный сторонний запрос — опциональная проверка IP выше, которая идёт через ваш активный прокси и раскрывает exit IP, но никогда — реальный.
- Не встраивает сторонние трекеры и библиотеки fingerprinting.
- Не требует регистрации или входа.
- Не пишет на диск за пределами каталога данных хелпера (конфиг sing-box + логи, которые вы можете прочитать сами).

## Какие разрешения запрашивает Noctis и зачем

| Разрешение | Для чего |
| --- | --- |
| `proxy` | Сказать Chrome использовать локальный SOCKS-слушатель sing-box, когда активен профиль маршрутизации. |
| `storage` | Сохранять ваши серверы, правила и настройки в браузере. |
| `nativeMessaging` | Единственный канал, по которому расширение общается с локальным хелпером. |
| `privacy` | Требуется Chrome proxy API для области настройки прокси. |
| `alarms` | Периодические health-проверки и обновление подписок по расписанию. |
| `tabs` | Side panel и контекст статуса по вкладкам. |
| `declarativeNetRequestWithHostAccess` | Блокировка ad/tracker доменов на уровне активного профиля маршрутизации. |
| `host_permissions: <all_urls>` | Правила маршрутизации сопоставляют произвольные адресаты; решения по конкретному URL требуют видимости URL. |

## Изменения политики

Значимые изменения публикуются здесь с новой датой «Обновлено». Листинг в магазине будет обновлён под текущую версию.
